Stop QGIS from Creating XML Files for Raster Layers

Have you ever noticed that QGIS creates an XML file every time you close a raster layer? If you're like me, and you deal with a lot of automated processing, this can be a real pain. Constantly having to filter out these extra XML files from your file lists is tedious.

This behavior is related to GDAL (Geospatial Data Abstraction Library), which QGIS uses for raster processing. Specifically, it's the creation of Persistent Auxiliary Metadata (PAM) files, typically named *.aux.xml. These files store metadata, including raster statistics, to speed up subsequent reads.

Here's how to stop QGIS from creating these XML files:

The Solution: Disabling GDAL PAM

The key is to disable GDAL's PAM functionality. Here's how:

  1. Open QGIS Settings: Go to Settings > Options.
  2. Navigate to System Environment: Select the System tab, then the Environment tab.
  3. Add a New Variable: Click the "+" button to add a new environment variable.
  4. Set the Variable:
    • Variable: GDAL_PAM_ENABLED
    • Value: NO
  5. Restart QGIS: For the changes to take effect, you'll need to close and reopen QGIS.

Important Notes:

  • GDAL's Role: The *.aux.xml files are created by GDAL, not directly by QGIS.
  • GDAL_PAM_PROXY_DIR: Setting GDAL_PAM_PROXY_DIR will not prevent the creation of *.aux.xml files if you have write permissions in the raster's directory. This variable is only relevant when you lack write permissions.
  • Statistics Caching: Disabling PAM will prevent GDAL from caching raster statistics. This might slightly slow down the display of rasters on subsequent openings, as statistics will need to be recalculated.

By following these steps, you can eliminate the unwanted XML files and streamline your raster processing workflow in QGIS.
